# Web Page Monitor

一个用于自我约束的 Chrome/Edge 扩展：受控网站可以同时按“允许时间段”和“禁止时间段”管理。触发限制时，需要填写打开原因，扩展会把时间、网站、地址、原因和放行时长记录到本地日志。

## 加载方式

1. 打开 Chrome 或 Edge 的扩展管理页。
2. 开启“开发者模式”。
3. 选择“加载已解压的扩展”。
4. 选择该项目所在的目录。
5. 点击扩展图标进入设置页。

## 两种时间策略

- 启用允许时间段：只有允许时间段内可以直接打开，其他时间需要填写原因。
- 启用禁止时间段：禁止时间段内需要填写原因，其他时间可以直接打开。

两套规则都有独立开关，可以只开一个、两个都开，或者两个都关。两个都开时，只要“不在允许时间段内”或“处于禁止时间段内”任一条件命中，就需要填写原因。

## 默认规则

- 受控网站：`youtube.com`、`bilibili.com`、`x.com`、`reddit.com`
- 默认生效方式：启用允许时间段，禁用禁止时间段
- 默认允许时间：
  - 周一到周五 `20:00-21:00`
  - 周六、周日 `14:00-15:30`
  - 周六、周日 `20:00-21:00`
- 触发时间限制后，默认放行 `10` 分钟。

## 数据存储

设置、临时放行和访问日志都保存在浏览器的 `chrome.storage.local`，不会上传到服务器。日志最多保留最近 1000 条，可以在设置页导出 CSV 或 JSON。

更多说明见 [PRIVACY.md](PRIVACY.md)。

## 开发检查

这个项目没有构建步骤，直接加载目录即可。分享或修改代码前，可以运行：

```bash
npm run check
npm test
```

核心时间规则在 `shared.js`，对应测试在 `tests/policy.test.js`。

## 文件结构

- `manifest.json`：Chrome/Edge 扩展声明
- `shared.js`：时间规则、域名匹配、设置归一化等共享逻辑
- `background.js`：监听导航、判断限制、写入日志
- `options.html` / `options.css` / `options.js`：设置页和日志页
- `blocked.html` / `blocked.css` / `blocked.js`：拦截页
- `tests/policy.test.js`：核心规则测试

## 边界

浏览器扩展适合制造提醒、摩擦和记录。它不能防止你卸载扩展、换浏览器或换设备。如果以后想提高强度，可以叠加浏览器企业策略、系统 hosts、防火墙、DNS 或路由器规则。
